## Technical Context

### Routes

```
GET /api/graph                     â†’ JSON graph data (all domains)
GET /api/graph?domain=A,B          â†’ JSON graph data filtered to domain A or B
GET /graph                         â†’ serves GRAPH_HTML (SSR shell, client-side render)
```

### API Response Shape

```json
{
  "nodes": [
    { "id": "uuid", "name": "supply and demand", "display_name": "Supply and Demand", "domain": "Economics" }
  ],
  "edges": [
    { "source": "uuid1", "target": "uuid2", "relation_type": "related", "strength": 0.8 }
  ]
}
```

### Domain Filtering Behavior

- The `?domain=` query parameter is a comma-separated list of domain names (case-sensitive).
- When specified, the backend filters `concept` nodes to those matching any of the listed domains.
- Edges are filtered post-query: only edges whose both `source` and `target` are in the filtered node set are returned.
- The frontend performs **client-side** filtering from the full graph payload (all nodes loaded once; chips toggle `activeDomains` state and re-run `applyFilter()`).
- The `?domain=` server-side API is available for programmatic consumers.

### Handler Location

- `src/web/handlers/graph.rs` â€” `get_api_graph` (handles `?domain=` query param)
- `src/adapters/surreal/graph_read_repo.rs` â€” `SurrealGraphReadRepo::get_graph`

### UI Implementation

- Domain chips are built dynamically in `buildDomainChips()` from `uniqueDomains`, which is populated from node data on initial load.
- Chip state (`activeDomains` Set) drives `applyFilter()`, which rebuilds the visible `nodes` and `edges` arrays and re-runs the force simulation.
- The "All" chip clears `activeDomains`; individual domain chips toggle membership.
- ARIA: chips use `role="checkbox"` + `aria-checked` toggled on each click.

---

## Implementation Notes

- Domain names are free-form strings (OQ-06) â€” filtering is case-sensitive and exact-match.
- The backend SQL filter uses `WHERE domain IN $domains` with parameterized binding to avoid injection.
- Edge filtering in `get_graph` is done in Rust after fetching all edges: only edges with both endpoints in `node_ids` are kept when a domain filter is active.
- No separate `/api/graph/domains` endpoint â€” the frontend derives unique domains from the graph response.
