## User Story

**As Beatriz**, I want to click on a concept node and see all books that mention it plus its related concepts, so that I can trace ideas across my reading history.

---

## Acceptance Criteria

```
Given I click on a concept node in the graph
When I am redirected to /graph/concepts/{id}
Then I see: concept name, domain badge, description, a list of books that
  mention it (via menciona ← interpreta ← work traversal), and its related
  concepts with relation types and strength

Given I am on the concept detail page
When the page loads
Then it fetches GET /api/concepts/{id} and renders the data without a full
  page reload (client-side JS fetch)

Given the concept has related concepts
When I click a related concept card
Then I navigate to that concept's detail page at /graph/concepts/{id}

Given I am on the concept detail page
When I click a book title in the "Books" section
Then I navigate to /works/{work_id}

Given I am not authenticated
When I access /graph/concepts/{id}
Then I am redirected to /login (or /setup if no admin exists)

Given a concept_id does not exist
When GET /api/concepts/{id} is called
Then the API returns HTTP 404 and the page renders an error state

Given the API call fails
When an error occurs fetching concept data
Then the page renders an accessible error message (role="alert")
```

## Technical Context

### Routes

```
GET /graph/concepts/{id}   → serves CONCEPT_DETAIL_HTML (SSR shell + client-side fetch)
GET /api/concepts/{id}     → JSON concept detail (auth-protected)
```

### API Response Shape

```json
{
  "concept": {
    "id": "concept:abc123",
    "name": "machine learning",
    "display_name": "Machine Learning",
    "description": "A branch of artificial intelligence...",
    "domain": "Computer Science"
  },
  "books": [
    {
      "work_id": "work:xyz",
      "title": "The Pragmatic Programmer",
      "author": "David Thomas"
    }
  ],
  "related_concepts": [
    {
      "id": "concept:def456",
      "display_name": "Deep Learning",
      "domain": "Computer Science",
      "relation_type": "extends",
      "strength": 0.85
    }
  ]
}
```

### Domain Types (src/domain/concept.rs)

```rust
pub struct ConceptDetail {
    pub concept: Concept,

    pub related_concepts: Vec<RelatedConceptSummary>,

pub struct ConceptBook {

pub struct RelatedConceptSummary {
    pub id: String,
    pub display_name: String,
    pub domain: String,
    pub relation_type: String,

}
```

### GraphReadRepo Port (src/ports/repository.rs)

```rust
async fn get_concept_detail(id: &str) -> Result<Option<ConceptDetail>, RepoError>;
```

### SurrealDB Queries

**Concept lookup:**
```surrealql
SELECT *, meta::id(id) AS id FROM type::thing('concept', $id)
```

**Books mentioning concept:**
```surrealql
SELECT in AS insight_id FROM menciona WHERE out = type::thing('concept', $id)
-- then:
SELECT in AS work_id, in.title AS title, in.author AS author
  FROM interpreta WHERE out IN $insight_ids
```

**Related concepts:**
```surrealql
SELECT out AS concept_id, out.display_name AS display_name,
  out.domain AS domain, relation_type, strength
  FROM relacionado_a WHERE in = type::thing('concept', $id)
```

### Handler Locations

- `src/web/handlers/graph.rs` — `get_api_concept_by_id` and `get_concept_detail_page`
- `src/web/router.rs` — routes `/api/concepts/{id}` and `/graph/concepts/{id}`

### UI Implementation

The concept detail page is an SSR-served HTML shell that uses client-side JS to:
1. Extract the concept ID from `window.location.pathname`
2. Fetch `/api/concepts/{id}` with `credentials: 'same-origin'`
3. Render concept name, domain badge, description, books list, related concepts grid
4. Update `document.title` and breadcrumb dynamically

### Navigation from Graph

The graph page (`/graph`) navigates to `/graph/concepts/{id}` on node click via:
```js
window.location.href = '/graph/concepts/' + encodeURIComponent(hit.id);
```

---

## Implementation Notes

- The concept ID from the URL is SurrealDB record ID (e.g., `concept:abc123`). The API strips the `concept:` prefix before querying so `encodeURIComponent` on the full ID is safe.
- Strength values (0.0–1.0) are rendered as a visual bar using a `<span>` with `width: X%` inline style (dynamic value, not a design token — the only acceptable exception per architecture).
- `escapeHtml` sanitizes all API-returned strings to prevent XSS.
- WCAG AA: `aria-live="polite"` on loading state, `role="alert"` on error state, `role="list"` on book list, focus rings on all interactive elements using BMW Focus Blue (`#0653b6`).
- The `back-link` returns to `/graph`; the breadcrumb shows `Graph › <concept name>`.
