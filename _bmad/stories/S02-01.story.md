## User Story

**As Beatriz**, I want to submit a book by ISBN, so that the system can look it up and start building my knowledge graph.

---

## Acceptance Criteria

```
Given I am authenticated (JWT or PAT)
When I submit a valid ISBN-10 or ISBN-13 via POST /api/works
  with body: {"identifier": "<isbn>", "identifier_type": "isbn"}
Then a work record is created with status = 'pending'
And the response is HTTP 202 with body: {"work_id": "<uuid>", "status": "pending"}
And a discovery.requested NATS event is published with the work_id

Given I submit an ISBN that already exists in the database
When the server processes the request
Then the response is HTTP 409 with body: {"work_id": "<existing_uuid>", "error": "duplicate"}
And no new work record is created
And no NATS event is published

Given I submit a malformed ISBN (wrong length or invalid check digit)
When the server processes the request
Then the response is HTTP 422 with body: {"error": "invalid_isbn", "message": "Invalid ISBN-13 — check digit mismatch"}
And no work record is created

Given I am not authenticated
When I call POST /api/works
Then the response is HTTP 401
```

## Technical Context

### Endpoint

```
POST /api/works
Authorization: Bearer <jwt_or_pat>
Content-Type: application/json

Body:
{
  "identifier": "9780132350884",
  "identifier_type": "isbn"
}

Responses:
  202: {"work_id": "uuid", "status": "pending"}
  409: {"work_id": "existing_uuid", "error": "duplicate"}
  422: {"error": "invalid_isbn", "message": "..."}
  401: {"error": "unauthorized", "message": "..."}
```

### Handler location

`src/web/handlers/works.rs` — `post_works` handler.

### Work record schema (SurrealDB)

```surrealql
-- Table already defined in schema.rs (startup migration):
-- work: id, title, author, isbn, open_library_id, status, error_msg, created_at, updated_at
-- Unique index on isbn: DEFINE INDEX IF NOT EXISTS work_isbn ON work COLUMNS isbn UNIQUE

INSERT INTO work {
  id: rand::uuid(),
  title: "",          -- populated by worker after Open Library fetch
  author: "",         -- populated by worker after Open Library fetch
  isbn: $isbn,
  open_library_id: NONE,
  status: "pending",
  error_msg: NONE,
  created_at: time::now(),
  updated_at: time::now()
}
```

### Deduplication logic

Check for existing work by ISBN before inserting:
```surrealql
SELECT id FROM work WHERE isbn = $isbn LIMIT 1
```
If a record is found, return 409 immediately — do not insert, do not publish to NATS.

Note: `open_library_id` deduplication is handled by S02-02 (title submission). This story only needs ISBN deduplication.

### NATS event

After successful work creation, publish to JetStream subject `discovery.requested`:
```json
{
  "work_id": "<uuid>",
  "identifier": "<isbn>",
  "identifier_type": "isbn"
}
```

Use the `MessagePublisher` port trait (`src/ports/messaging.rs`). The NATS adapter must be wired into `AppState` before this story can integrate end-to-end. If NATS is not yet initialized, return a 500 (do not silently swallow the publish error).

### ISBN validation

Validate in the domain layer (`src/domain/work.rs`), not in the handler. Two checks required:

**ISBN-13:**
- Must be exactly 13 digits (strip hyphens first)
- Check digit: multiply alternating digits by 1 and 3, sum, check that `sum % 10 == 0`

**ISBN-10:**
- Must be exactly 10 characters (digits, last char may be 'X')
- Check digit: `sum(d[i] * (10 - i)) % 11 == 0` where X = 10

Return a domain error (`WorkError::InvalidIsbn(message)`) on failure — the handler maps this to HTTP 422.

### Auth middleware

This endpoint is protected by the existing JWT/PAT middleware (FR-06). No special auth logic needed in this handler — the middleware rejects unauthenticated requests before the handler runs.

### Dependencies

- DB schema must be initialized (startup migration in `src/adapters/surreal/schema.rs`)
- `WorkRepo` trait and `SurrealWorkRepo` implementation must exist (`src/adapters/surreal/work_repo.rs`)
- `MessagePublisher` trait and `NatsPublisher` implementation must exist (`src/adapters/nats/publisher.rs`)
- Auth middleware must be active on `/api/*` routes
- **This story does NOT depend on S02-02 or any worker story** — it only creates the work record and publishes the event. The worker consumes the event independently.

## Implementation Notes

### Error classification

| Case | HTTP | Domain error |
|------|------|-------------|
| Malformed ISBN | 422 | `WorkError::InvalidIsbn` |
| Duplicate ISBN | 409 | `WorkError::Duplicate { work_id }` |
| NATS publish failure | 500 | `WorkError::MessagingError` |
| DB write failure | 500 | `WorkError::DatabaseError` |

### Logging

Every request to this handler must log at INFO level with the `work_id` in the span context on success:
```
INFO work_id=<uuid> isbn=<isbn> msg="work created, discovery.requested published"
```

On duplicate:
```
INFO isbn=<isbn> existing_work_id=<uuid> msg="duplicate ISBN, returning 409"
```

### Testing

**Unit tests** (`src/domain/work.rs`):
- Valid ISBN-13 accepted
- Invalid ISBN-13 check digit rejected with correct message
- Valid ISBN-10 accepted (including X check digit)
- Invalid ISBN-10 rejected
- ISBN with hyphens normalized before validation

**Integration tests** (`tests/integration/surreal_work_repo.rs`):
- `create_work` inserts a record and returns the work_id
- `find_by_isbn` returns Some for existing ISBN, None otherwise
- Unique index on isbn prevents duplicate inserts at DB level

**Handler tests** (via `axum::test`):
- POST with valid ISBN → 202
- POST with duplicate ISBN → 409 with existing work_id
- POST with invalid ISBN → 422
- POST without auth → 401
